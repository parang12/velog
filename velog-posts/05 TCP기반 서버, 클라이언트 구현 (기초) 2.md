<h2 id="클라이언트-보강">클라이언트 보강</h2>
<p>앞서 배웠던 04장의 <code>echo_client</code> 가 사실은 문제가 있는 코드이다. 그 이유는
 -&gt; read(), write()가 호출될 때 마다 문자열 단위로 입출력이 일어난 다는 것이다</p>
<pre><code class="language-c"> write(sock, message, strlen(massage));
 str_len = read(sock, message, BUF_SIZE - 1);
 message[str_len] = 0;
 printf(&quot;Message from server: %s&quot;, message);</code></pre>
<p>[02 소켓의 타입 및 프로토콜]에서 다뤘듯이, TCP클라이언트는 둘 이상의 wirte함수를 통해서 문자열 정보가 묶여 <strong>한번에</strong> 서버로 전송 될 수 있다. 그렇게 된다면 클라이언트는 둘 이상의 문자열 정보를 서버로 부터 되돌려 받아서 원하는 결과를 얻지 못 할 수도 있다.</p>
<p>서버는 한번의 write 함수로 데이터를 전송 명령을 했지만, 전송할 데이터의 크기가 크다면 운영체제는 내부적으로 이를 여러개의 조각으로 나눠서 클라이언트에 전송할 수 도 있기 때문이다. 또한 여러개의 조각이 클라이언트에 전송되지 않았음에도 불구하고 클라이언트는 read함수를 호출할지도 모른다는 위험이 있다.</p>
<pre><code class="language-text">TCP는 데이터의 경계가 없기 때문
오류 1
    write(&quot;ABC&quot;)
    wirte(&quot;DEF&quot;)
    이런식으로 보내게 되면

    read() 호출 -&gt; &quot;ABCDEF&quot; 

TCP는 '바이트 스트림' 기반이기 때문에, write 한 번이 read 한 번을 보장하지 않음
오류 2
    write(&quot;Very long message...&quot;) -&gt; 매우큰 데이터 전송
    네트워크에 따라서 여러개의 세그먼크로 쪼개서 보냄


    다 도착하지 않은 상태 read() 호출 -&gt; 전체 매세지의 첫부분만 읽고 끝나버림</code></pre>

<h2 id="tcp-이론적-이야기">TCP 이론적 이야기</h2>
<h3 id="tcp-소켓의-버퍼">TCP 소켓의 버퍼</h3>
<p>TCP는 write함수를 쓴다고 데이터가 전송되지 않으며, 마찬가지로 read함수를 쓰더라도 데이터가 수신되는것이 아니다. </p>
<ul>
<li>write함수 -&gt; 소켓 출력버퍼 데이터 전달</li>
<li>read함수 -&gt; 소켓 입력버퍼 저장된 데이터 읽기</li>
</ul>
<p>또한 TCP에서는 슬라이딩 윈도우(Sliding)라는 프로토콜이 존재하기 때문에 입력버퍼를 초과하는 분량의 데이터 전송은 발생하지 않음.</p>

<h2 id="애플리케이션-프로토콜-설계-예제">애플리케이션 프로토콜 설계 예제</h2>
<h3 id="계산기-서버쪽-코드드">계산기 서버쪽 코드드</h3>
<pre><code class="language-c">#include //이전내용과 같음

#define BUF_SIZE 1024
#define OPSZ 4
void error_handling(char *message);
int calculate(int opnum, int opnds[], char operator);

int main(int argc, char *argv[])
{
    int serv_sock, clnt_sock;
    char opinfo[BUF_SIZE];
    int result, opnd_cnt, i;
    int recv_cnt, recv_len;
    struct sockaddr_in serv_adr, clnt_adr;
    socklen_t clnt_adr_sz;

    /* ... 소켓 생성, bind, listen 과정 생략 (에코 서버와 동일) ... */

    for(i=0; i&lt;5; i++)
    {
        opnd_cnt = 0;
        clnt_sock = accept(serv_sock, (struct sockaddr*)&amp;clnt_adr, &amp;clnt_adr_sz);

        // [프로토콜의 구현] 
        // 약속된 프로토콜에 따라 첫 1바이트(피연산자 개수)를 먼저 읽습니다.
        read(clnt_sock, &amp;opnd_cnt, 1);

        recv_len = 0;
        // [데이터 분해 - 바이트 배열 수신]
        // 프로토콜 규칙: (개수 * 4바이트) + 마지막 연산자 1바이트만큼 데이터를 더 읽어야 함
        // TCP의 데이터 경계 없음을 극복하기 위해 목표치까지 반복해서 읽습니다.
        while((opnd_cnt * OPSZ + 1) &gt; recv_len)
        {
            recv_cnt = read(clnt_sock, &amp;opinfo[recv_len], BUF_SIZE - 1);
            recv_len += recv_cnt;
        }

        // [데이터 해석(Parsing) 및 처리]
        // 수신된 바이트 뭉치(opinfo)를 프로토콜 규칙에 따라 실제 숫자와 기호로 분리하여 계산합니다.
        result = calculate(opnd_cnt, (int*)opinfo, opinfo[recv_len - 1]);

        // 결과값 전송
        write(clnt_sock, (char*)&amp;result, sizeof(result));
        close(clnt_sock);
    }
    close(serv_sock);
    return 0;
}

int calculate(int opnum, int opnds[], char op)
{
    int result = opnds[0], i;
    switch(op)
    {
        case '+':
            for(i=1; i&lt;opnum; i++) result += opnds[i];
            break;
        case '-':
            for(i=1; i&lt;opnum; i++) result -= opnds[i];
            break;
        case '*':
            for(i=1; i&lt;opnum; i++) result *= opnds[i];
            break;
    }
    return result;
}

void error_handling(char *message) { /* 생략 */ }</code></pre>
<h3 id="계산기-클라이언트-코드">계산기 클라이언트 코드</h3>
<pre><code class="language-c">#include //이전과 같음음
#define BUF_SIZE 1024
#define RLT_SIZE 4
#define OPSZ 4
void error_handling(char *message);

int main(int argc, char *argv[])
{
    int sock;
    char opinfo[BUF_SIZE]; // 서버로 보낼 데이터를 조립할 바이트 배열(버퍼)
    int result, opnd_cnt, i;
    struct sockaddr_in serv_adr;

    /* ... 소켓 생성 및 connect 과정 생략 (기본 클라이언트와 동일) ... */

    fputs(&quot;Operand count: &quot;, stdout);
    scanf(&quot;%d&quot;, &amp;opnd_cnt);

    // [프로토콜 설계 - 헤더(Header) 구성]
    // 약속된 프로토콜에 따라 첫 번째 바이트에 피연산자의 개수 정보를 담습니다.
    opinfo[0] = (char)opnd_cnt;

    // [데이터 직렬화(Serialization) - 데이터 조립]
    // 정수(int) 데이터를 바이트 배열인 opinfo의 적절한 위치에 '강제로' 끼워 넣습니다.
    for(i=0; i&lt;opnd_cnt; i++)
    {
        printf(&quot;Operand %d: &quot;, i+1);
        // 포인터 조작을 통해 4바이트 int 값을 바이트 배열에 직접 복사합니다.
        // i*OPSZ + 1 위치부터 4바이트 공간을 int의 주소로 해석하여 저장하는 방식입니다.
        scanf(&quot;%d&quot;, (int*)&amp;opinfo[i*OPSZ + 1]);
    }

    fgetc(stdin); // 입력 버퍼에 남아있는 개행 문자(\n)를 제거하여 다음 scanf 오류 방지

    fputs(&quot;Operator: &quot;, stdout);
    // 피연산자 데이터들이 끝난 바로 다음 위치에 1바이트 연산자 정보를 추가합니다.
    scanf(&quot;%c&quot;, &amp;opinfo[opnd_cnt * OPSZ + 1]);

    // [데이터 전송 - 규격화된 패킷 전송]
    // 약속한 프로토콜의 총 크기(개수 1 + 데이터 n*4 + 연산자 1)를 계산해서 보냅니다.
    write(sock, opinfo, opnd_cnt * OPSZ + 2); 

    // [데이터 수신 및 해석]
    // 서버가 계산 후 보내줄 4바이트 결과값(int)을 그대로 읽어 들입니다.
    read(sock, &amp;result, RLT_SIZE); 

    printf(&quot;Operation result: %d \n&quot;, result);

    close(sock);
    return 0;
}

void error_handling(char *message) { /* 생략 */ }</code></pre>
<ul>
<li><strong>사용자 정의 프로토콜:</strong> TCP가 제공하는 기본 통신 위에,  직접 데이터 구조(헤더, 바디)를 설계함.</li>
<li><strong>데이터 바인딩:</strong> 정수(<code>int</code>)와 문자(<code>char</code>)를 바이트 배열의 특정 위치에 메모리 주소 단위로 때려 박는 방식.</li>
<li><strong>동적 패킷 생성:</strong> 고정된 크기가 아니라 사용자가 입력한 개수에 따라 전송할 데이터의 크기가 실시간으로 변하는 구조.</li>
</ul>

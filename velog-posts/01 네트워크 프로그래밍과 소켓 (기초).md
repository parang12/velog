## 네트워크 프로그래밍
 -> 네트워크로 연결되어있는 두 컴퓨터가 데이터를 주고 받을 수 있도록 하는 것
 - 물리적 연결[유선] : 네트워크 카드 & 케이블 이용
 - 논리적 연결[무선] : 소켓 이용

> 소켓: 네트워크상에서 통신하는 두 프로그램 사이의 양방향 통신의 종착점(End-point)이자, 운영체제가 제공하는 추상화된 인터페이스(API)이다.

(서버: Server) <----------------------> (클라이언트: Client)

- 서버: 연결요청을 수락하는 기능 (주요 단계: Bind -> Listen -> Accept)
- 클라이언트: 연결 요청을 거는 기능 (주요 단계: Connect)

---
대표적인 소켓 2가지: TCP, UDP
이중에서 처음에 다뤄볼 소켓은 `TCP`이다.

## TCP

>TCP(Transmission Control Protocl): 전송 제어 프로토콜이라고 부르며, 전송 계층에서 **연결 지향적이고 신뢰성 있는 데이터 전송**을 보장하는 프로토콜이다. 

```text
📃tip
인터페이스: 논리/물리적 연결하는 방식.
프로토콜: 통신 규칙, 규약.
```


## 예제

- 서버 코드
```C
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h> 
#include <arpa/inet.h>
#include <sys/socket.h>

void error_handling(char *message);

int main(int argc, char **argv) {
    int serv_sock; 
    int clnt_sock; 

    struct sockaddr_in serv_addr; 
    struct sockaddr_in clnt_addr; 
    socklen_t clnt_addr_size; 

    char message[] = "Hello World!\n";

    if (argc != 2) { 
        printf("Usage: %s <port>\n", argv[0]);
        exit(1);
    }

    /* -----------------------------------------------------------
     * [STEP 1] 소켓 생성 (Socket Creation)
     * : 운영체제로부터 네트워크 통신에 필요한 자원(소켓)을 할당받습니다.
     * ----------------------------------------------------------- */
    serv_sock = socket(PF_INET, SOCK_STREAM, 0);
    if (serv_sock == -1) 
        error_handling("socket() error");

    /* -----------------------------------------------------------
     * [STEP 2] 주소 및 포트 설정 (Address Setting)
     * : 서버가 사용할 IP 주소와 포트 번호를 구조체에 기록합니다.
     * ----------------------------------------------------------- */
    memset(&serv_addr, 0, sizeof(serv_addr));
    serv_addr.sin_family = AF_INET;
    serv_addr.sin_addr.s_addr = htonl(INADDR_ANY); 
    serv_addr.sin_port = htons(atoi(argv[1]));     

    /* -----------------------------------------------------------
     * [STEP 3] 소켓 주소 할당 (Bind)
     * : 위에서 설정한 주소 정보를 생성한 소켓에 실제로 입힙니다.
     * ----------------------------------------------------------- */
    if (bind(serv_sock, (struct sockaddr*)&serv_addr, sizeof(serv_addr)) == -1)
        error_handling("bind() error");
   
    /* -----------------------------------------------------------
     * [STEP 4] 연결 요청 대기 (Listen)
     * : 이제 클라이언트가 접속할 수 있도록 소켓을 대기 상태로 만듭니다.
     * ----------------------------------------------------------- */
    if (listen(serv_sock, 5) == -1)
        error_handling("listen() error");

    /* -----------------------------------------------------------
     * [STEP 5] 연결 수락 및 통신 소켓 생성 (Accept)
     * : 실제 접속한 클라이언트와 통신할 전용 소켓을 새롭게 생성합니다.
     * ----------------------------------------------------------- */
    clnt_addr_size = sizeof(clnt_addr);
    clnt_sock = accept(serv_sock, (struct sockaddr*)&clnt_addr, &clnt_addr_size);
    if (clnt_sock == -1)
        error_handling("accept() error");

    /* -----------------------------------------------------------
     * [STEP 6] 데이터 송수신 (Write/Read)
     * : 연결된 소켓을 통해 실제로 데이터를 주고받습니다.
     * ----------------------------------------------------------- */
    write(clnt_sock, message, sizeof(message));

    /* -----------------------------------------------------------
     * [STEP 7] 소켓 폐쇄 (Close)
     * : 통신이 끝난 소켓 자원을 운영체제에 반환하고 종료합니다.
     * ----------------------------------------------------------- */
    close(clnt_sock);
    close(serv_sock); 

    return 0;
}

void error_handling(char *message) {
    fputs(message, stderr);
    fputc('\n', stderr);
    exit(1);
}
```

- 클라이언트 코드

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <arpa/inet.h>
#include <sys/socket.h>

void error_handling(char *message);

int main(int argc, char *argv[]) {
    int sock;
    struct sockaddr_in serv_addr;
    char message[30];
    int str_len;

    if (argc != 3) exit(1);

    /* -----------------------------------------------------------
     * [STEP 1] 소켓 생성 (Socket Creation)
     * ----------------------------------------------------------- */
    sock = socket(PF_INET, SOCK_STREAM, 0);

    /* -----------------------------------------------------------
     * [STEP 2] 대상 서버 주소 설정 (Server Address Setting)
     * ----------------------------------------------------------- */
    memset(&serv_addr, 0, sizeof(serv_addr));
    serv_addr.sin_family = AF_INET;
    serv_addr.sin_addr.s_addr = inet_addr(argv[1]);
    serv_addr.sin_port = htons(atoi(argv[2]));

    /* -----------------------------------------------------------
     * [STEP 3] 연결 요청 (Connect)
     * ----------------------------------------------------------- */
    if(connect(sock, (struct sockaddr*)&serv_addr, sizeof(serv_addr)) == -1)
        error_handling("connect() error!");

    /* -----------------------------------------------------------
     * [STEP 4] 데이터 수신 (Read)
     * ----------------------------------------------------------- */
    str_len = read(sock, message, sizeof(message) - 1);
    if(str_len==-1)
        error_handling("read() error!");
    /* -----------------------------------------------------------
     * [STEP 5] 소켓 폐쇄 (Close)
     * ----------------------------------------------------------- */
    printf("Message from server : %s\n", message);
    close(sock);

    return 0;
}

void error_handling(char *message) {
    fputs(message, stderr);
    fputc('\n', stderr);
    exit(1);
}
```

## 결론

| **단계** | **서버 (Server)**   | **상호작용** | **클라이언트 (Client)**   |
| ------ | ----------------- | -------- | -------------------- |
| **대기** | `listen()` (문 열기) | →        | (대기실 생성됨)            |
| **연결** | `accept()` (기다림)  | ←        | `connect()` (문 두드리기) |
| **통신** | `write()` (말하기)   | →        | `read()` (듣기)        |
| **종료** | `close()` (작별)    | ↔        | `close()` (작별)       |

>"서버 프로그래밍에서 가장 중요한 것은 **함수의 호출 순서**입니다. 서버는 `listen()`을 통해 '대기실'을 먼저 만들어야 하며, `accept()` 함수는 클라이언트가 문을 두드릴 때까지 프로세스를 멈추고 기다리는 **블로킹(Blocking)** 특성을 가집니다."

<p>※본 내용은 열혈 TCP/IP 내용을 참고 하였으며, 리눅스 환경 기반으로 테스트를 진행하였습니다.</p>
<h2 id="ip-주소">IP 주소</h2>
<p>인터넷에 컴퓨터를 연결해서 데이터를 주고 받기 위해서는 IP주소를 부여 받아야 한다. 대표적으로 2가지가 있으며 종류는 다음과 같다</p>
<ul>
<li>IPv4 (Internet Protocol version 4) &lt;- 4byte</li>
<li>IPv6 (Internet Protocol version 6) &lt;- 16byte</li>
</ul>
<p>왜? 2개나 있을까? -&gt; IPv4가 나왔으나 전세계 인구 증가 및 전자기기의 다양화로 IPv4주소 고갈위기가 되자 IPv6라는 것을 내놓았다.
하지만 오늘날 우리는 거의 대부분 IPv4를 사용하고 있다. (NTA덕분에 가능)</p>
<p><img alt="" src="https://velog.velcdn.com/images/parang12/post/05f25324-9a76-4064-b42d-172de0c4de3b/image.png" /></p>

<p>5개의 클래스로 나뉜 이유</p>
<ol>
<li>서로 다른 규모의 조직에 적절한 양의 주소를 주기 위해</li>
<li>라우터가 주소를 아주 빠르게 읽을 수 있도록 하기 위해</li>
</ol>

<h2 id="포트번호">포트번호</h2>
<p>포트 번호는 하나의 컴퓨터 내에서 실행되고 있는 특정 프로그램(프로세스)을 식별하기 위한 번호이다.</p>
<p><code>IP가 주소라라면, 포트번호는 상세주소라고 생각하면 이해가 쉽다</code></p>
<p>우리가 평소에는 잘 못 보지만, 사실 모든 주소 뒤에는 포트 번호가 숨어 있다.
    -&gt; <code>https://google.com</code> (실제로는 <code>https://google.com:443</code>)</p>
<p>따라서 주소정보는 IP + 포트번호로 이루어져 있고 이걸 OS에서 받아 그것에 해당하는 소켓에 연결시켜 데이터가 전달된다고 보면 된다.</p>
<pre><code class="language-Text">IP + 포트번호 -&gt; OS가 받아서 그것에 해당하는 소켓에 연결</code></pre>
<hr />
<h2 id="주소-정보의-표현">주소 정보의 표현</h2>
<p>어떤 주소를 사용? -&gt; IPv4기반
IP주소가 어떻게 됨 -&gt; 211.204.214.76 
PORT 번호는 어떻게됨? -&gt; 2048</p>
<p>이제 위 주소를 구조체 형태로 고쳐보자</p>
<pre><code class="language-c">struct sockaddr_in
{
    sa_family_t    sin_family; //주소체계, 이 주소가 IPv4인지 알려주는 값
    uint16_t       sin_port; //16비트 TCP/UDP PORT 번호
    struct in_addr sin_addr; //32비트 IPv4 주소
    char           sin_zero[8]; //구조체 크기 맞추기 위한 패딩, 사용안함
};</code></pre>
<p>sockaddr_in (소켓 내부) 구조체 안에 있는 in_addr이라는 구조체 내부 모습은 아래와 같다</p>
<pre><code class="language-c">struct in_addr
{
    in_addr_t    s_addr; // 32bit IPv4 인터넷 주소    
}</code></pre>
<p>이제 구조체의 변수들에 대해서 살펴보자</p>

<h3 id="sin_port">sin_port</h3>
<p>-&gt; 16비트 포트 번호를 저장한다 
네트워크 바이트 순서로 저장해야함 </p>
<pre><code>e.g.
32비트 숫자: 0x12345678 -&gt; 4바이트 변환: 12 34 56 78
메모리 앞쪽부터 큰 값부터 넣자 (빅 엔디안 MSB) 12 34 56 78
메모리 앞쪽부터 작은 값부터 넣자 (리틀 엔디안 LSB) 78 56 34 12

(크고 작은지는 가중치에 따라서)
[ 0x 12 34 56 78 ] 
      ↑       ↑ 
    큰 놈   작은 놈 
    (MSB)   (LSB)
-------------------------------------------------------
네트워크 바이트 순서 = 빅 엔디안 
CPU 기준 = 리틀 엔디안

따라서 네트워크가 통일해서 쓰는 빅 엔지안 순서로 넣어야 한다</code></pre><h3 id="sin_addr">sin_addr</h3>
<p>32비트 IP주소를 저장한다
이것도 네트워크 바이트 순서로 저장해야함.</p>
<h3 id="sin_zero">sin_zero</h3>
<p>특별한 의미를 지니지 않는 멤버, sockaddr_in의 크기를 구조체 sockaddr과 일치시키기 위해 삽입된 멤버이다. 
반드시 0으로 채워야 한다.</p>
<h2 id="ip주소가-32비트-정수로-변환되는-과정">IP주소가 32비트 정수로 변환되는 과정</h2>
<pre><code class="language-c">#include &lt;stdio.h&gt;
#include &lt;stdlib.h&gt;
#include &lt;arpa/inet.h&gt; // internet
void error_handling(char *message);
int main(int argc, char *argv[])
{
    char *addr = &quot;127.0.0.1&quot;; // ip주소
    struct sockaddr_in addr_inet; // 소켓 주소 구조체
    memset(&amp;addr_inet, 0, sizeof(addr_inet)); //사용전 초기화
    /*inet_aton() 함수는 문자열로 된 IP 주소를 네트워크 바이트 순서의 32비트
    * 정수로 변환하여 구조체에 저장한다.
    * addr = 우리가 선언했던 ip주소,
    * &amp;addr_inet.sin_addr = inet_aton() 함수가 변환한 네트워크 바이트 순서
    * 의 32비트 정수를 저장할 구조체의 멤버 */
    if (inet_aton(addr, &amp;addr_inet.sin_addr) == 0)
        error_handling(&quot;invalid address&quot;);
    else
        // 네트워크 바이트 순서의 32비트 정수 출력
        printf(&quot;Network ordered integer addr: %#x \n&quot;, addr_inet.sin_addr.s_addr); 
        return 0;
}
void error_handling(char *message)
{
    fputs(message, stderr);
    fputc('\n', stderr);
    exit(1);
}</code></pre>

<h2 id="32비트-주소체계를-ip주소로-변환-예제">32비트 주소체계를 IP주소로 변환 예제</h2>
<pre><code class="language-c">#include &lt;string.h&gt;
#include &lt;stdio.h&gt;
#include &lt;arpa/inet.h&gt; 

int main(int argc, char *argv[])
{
    struct sockaddr_in addr1, addr2; // 소켓 주소 구조체 생성
    char *str_ptr; 
    char str_arr[20];

    addr1.sin_addr.s_addr = htonl(0x1020304);
    addr2.sin_addr.s_addr = htonl(0x1010101);

    // inet_ntoa() 함수는 네트워크 바이트 순서의 32비트 정수를 점으로 구분된 4개의 10진수로 변환하여 
    // 문자열로 반환
    str_ptr = inet_ntoa(addr1.sin_addr); 

     // str_ptr이 가리키는 문자열을 str_arr로 복사 
    // inet_ntoa() 함수를 실행하면 str_ptr이 가리키는 문자열이 변경 되기 때문에 
    strcpy(str_arr, str_ptr);
    printf(&quot;Dotted-Decimal notation1: %s \n&quot;, str_ptr);

    inet_ntoa(addr2.sin_addr);
    printf(&quot;Dotted-Decimal notation2: %s \n&quot;, str_ptr);// str_ptr이 가리키는 문자열은 addr2의 IP 주소로 변경됨 
    printf(&quot;Dotted-Decimal notation3: %s \n&quot;, str_arr); 
    return 0;
}</code></pre>

<h2 id="요약">요약</h2>
<h3 id="-ip-주소-네트워크-상의-호스트-식별">### IP 주소: 네트워크 상의 호스트 식별</h3>
<ul>
<li><strong>종류</strong>:<ul>
<li><strong>IPv4 (4바이트)</strong>: 현재 주력 주소 체계. 고갈 위기를 <strong>NAT</strong>(Network Address Translation) 기술로 극복 중.</li>
<li><strong>IPv6 (16바이트)</strong>: IPv4 주소 부족 문제를 근본적으로 해결하기 위해 등장한 차세대 체계.</li>
</ul>
</li>
<li><strong>클래스 분류 (A~E)</strong>:<ul>
<li><strong>목적</strong>: 조직 규모에 맞는 효율적 주소 할당 + 라우터의 빠른 주소 판독.</li>
<li><strong>핵심 구분</strong>: 첫 번째 바이트(8비트)의 <strong>시작 비트</strong> 모양새로 구분함.</li>
</ul>
</li>
</ul>
<h3 id="포트-번호-port-프로세스-식별">포트 번호 (PORT): 프로세스 식별</h3>
<ul>
<li><strong>정의</strong>: 하나의 IP(컴퓨터) 내에서 실행 중인 특정 프로그램(소켓)을 구분하는 번호.</li>
<li><strong>특징</strong>: 모든 통신 주소 뒤에는 포트가 숨어 있음 (예: HTTPS = 443).</li>
<li><strong>데이터 흐름</strong>: <strong>IP + PORT</strong> 정보를 기반으로 OS가 해당 데이터를 적절한 <strong>소켓(Socket)</strong>에 전달함.</li>
</ul>
<h3 id="핵심-구조체-sockaddr_in">핵심 구조체: <code>sockaddr_in</code></h3>
<ul>
<li><strong>용도</strong>: IPv4 주소 정보를 담는 표준 규격 봉투.</li>
<li><strong>구성</strong>: 주소 체계(<code>sin_family</code>), 포트 번호(<code>sin_port</code>), IP 주소(<code>sin_addr</code>), 패딩(<code>sin_zero</code>).</li>
<li><strong>주의</strong>: <code>sin_zero</code>는 반드시 <strong>0으로 초기화</strong>해야 함 (구조체 크기 맞춤용).</li>
</ul>
<h3 id="바이트-순서-변환-endianness">바이트 순서 변환 (Endianness)</h3>
<ul>
<li><strong>Host(CPU)</strong>: 보통 리틀 엔디안 (하위 바이트부터 저장).</li>
<li><strong>Network</strong>: 반드시 <strong>빅 엔디안</strong> (상위 바이트부터 저장, 표준).</li>
<li><strong>변환 함수</strong>:<ul>
<li><code>htons</code> / <code>ntohs</code>: 2바이트 포트 번호용 ($s$ = short)</li>
<li><code>htonl</code> / <code>ntohl</code>: 4바이트 IP 주소용 ($l$ = long)        </li>
</ul>
</li>
</ul>
<h3 id="주소-변환-함수-문자열-↔-정수">주소 변환 함수 (문자열 ↔ 정수)</h3>
<ul>
<li><strong><code>inet_aton</code></strong>: 문자열 IP(&quot;127.0.0.1&quot;)를 네트워크 바이트 순서 정수로 변환 (성공 시 1 반환).</li>
<li><strong><code>inet_ntoa</code></strong>: 네트워크 정수를 다시 문자열로 변환.<ul>
<li><strong>경고</strong>: 내부 정적 버퍼를 사용하여 재호출 시 값이 덮어씌워짐. 즉시 <code>strcpy</code>로 복사 필수.</li>
</ul>
</li>
<li><strong>현대적 대안</strong>: IPv6 대응 및 Thread-safe를 위해 실무에선 <strong><code>inet_pton</code></strong>, <strong><code>inet_ntop</code></strong> 권장.※본 내용은 열혈 TCP/IP 내용을 참고 하였으며, 리눅스 환경 기반으로 테스트를 진행하였습니다.<h2 id="ip-주소-1">IP 주소</h2>
</li>
</ul>
<p>인터넷에 컴퓨터를 연결해서 데이터를 주고 받기 위해서는 IP주소를 부여 받아야 한다. 대표적으로 2가지가 있으며 종류는 다음과 같다</p>
<ul>
<li>IPv4 (Internet Protocol version 4) &lt;- 4byte</li>
<li>IPv6 (Internet Protocol version 6) &lt;- 16byte</li>
</ul>
<p>왜? 2개나 있을까? -&gt; IPv4가 나왔으나 전세계 인구 증가 및 전자기기의 다양화로 IPv4주소 고갈위기가 되자 IPv6라는 것을 내놓았다.
하지만 오늘날 우리는 거의 대부분 IPv4를 사용하고 있다. (NTA덕분에 가능)</p>

<h2 id="포트번호-1">포트번호</h2>
<p>포트 번호는 하나의 컴퓨터 내에서 실행되고 있는 특정 프로그램(프로세스)을 식별하기 위한 번호이다.</p>
<p><code>IP가 주소라라면, 포트번호는 상세주소라고 생각하면 이해가 쉽다</code></p>
<p>우리가 평소에는 잘 못 보지만, 사실 모든 주소 뒤에는 포트 번호가 숨어 있다.
    -&gt; <code>https://google.com</code> (실제로는 <code>https://google.com:443</code>)</p>
<p>따라서 주소정보는 IP + 포트번호로 이루어져 있고 이걸 OS에서 받아 그것에 해당하는 소켓에 연결시켜 데이터가 전달된다고 보면 된다.</p>
<pre><code class="language-Text">IP + 포트번호 -&gt; OS가 받아서 그것에 해당하는 소켓에 연결</code></pre>

<h2 id="주소-정보의-표현-1">주소 정보의 표현</h2>
<p>어떤 주소를 사용? -&gt; IPv4기반
IP주소가 어떻게 됨 -&gt; 211.204.214.76

<p>이제 위 주소를 구조체 형태로 고쳐보자</p>

    struct in_addr sin_addr; //32비트 IPv4 주소

<p>-&gt; 16비트 포트 번호를 저장한다

<pre><code>e.g.
32비트 숫자: 0x12345678 -&gt; 4바이트 변환: 12 34 56 78
메모리 앞쪽부터 큰 값부터 넣자 (빅 엔디안 MSB) 12 34 56 78

[ 0x 12 34 56 78 ]

    큰 놈   작은 놈 
    (MSB)   (LSB)
-------------------------------------------------------
네트워크 바이트 순서 = 빅 엔디안 
CPU 기준 = 리틀 엔디안

따라서 네트워크가 통일해서 쓰는 빅 엔지안 순서로 넣어야 한다</code></pre><h3 id="sin_addr-1">sin_addr</h3>

<h3 id="sin_zero-1">sin_zero</h3>

<h2 id="ip주소가-32비트-정수로-변환되는-과정-1">IP주소가 32비트 정수로 변환되는 과정</h2>
<pre><code class="language-c">#include &lt;stdio.h&gt;
#include &lt;stdlib.h&gt;
#include &lt;arpa/inet.h&gt; // internet
void error_handling(char *message);
int main(int argc, char *argv[])
{
    char *addr = &quot;127.0.0.1&quot;; // ip주소
    struct sockaddr_in addr_inet; // 소켓 주소 구조체
    memset(&amp;addr_inet, 0, sizeof(addr_inet)); //사용전 초기화
    /*inet_aton() 함수는 문자열로 된 IP 주소를 네트워크 바이트 순서의 32비트
    * 정수로 변환하여 구조체에 저장한다.
    * addr = 우리가 선언했던 ip주소,
    * &amp;addr_inet.sin_addr = inet_aton() 함수가 변환한 네트워크 바이트 순서
    * 의 32비트 정수를 저장할 구조체의 멤버 */
    if (inet_aton(addr, &amp;addr_inet.sin_addr) == 0)
        error_handling(&quot;invalid address&quot;);
    else
        // 네트워크 바이트 순서의 32비트 정수 출력
        printf(&quot;Network ordered integer addr: %#x \n&quot;, addr_inet.sin_addr.s_addr); 
        return 0;
}
void error_handling(char *message)
{
    fputs(message, stderr);
    fputc('\n', stderr);
    exit(1);
}</code></pre>

<h2 id="32비트-주소체계를-ip주소로-변환-예제-1">32비트 주소체계를 IP주소로 변환 예제</h2>
<pre><code class="language-c">#include &lt;string.h&gt;
#include &lt;stdio.h&gt;
#include &lt;arpa/inet.h&gt; 

int main(int argc, char *argv[])
{
    struct sockaddr_in addr1, addr2; // 소켓 주소 구조체 생성
    char *str_ptr; 
    char str_arr[20];

    addr1.sin_addr.s_addr = htonl(0x1020304);
    addr2.sin_addr.s_addr = htonl(0x1010101);

    // inet_ntoa() 함수는 네트워크 바이트 순서의 32비트 정수를 점으로 구분된 4개의 10진수로 변환하여 
    // 문자열로 반환
    str_ptr = inet_ntoa(addr1.sin_addr); 

     // str_ptr이 가리키는 문자열을 str_arr로 복사 
    // inet_ntoa() 함수를 실행하면 str_ptr이 가리키는 문자열이 변경 되기 때문에 
    strcpy(str_arr, str_ptr);
    printf(&quot;Dotted-Decimal notation1: %s \n&quot;, str_ptr);

    inet_ntoa(addr2.sin_addr);
    printf(&quot;Dotted-Decimal notation2: %s \n&quot;, str_ptr);// str_ptr이 가리키는 문자열은 addr2의 IP 주소로 변경됨

}</code></pre>

<h2 id="요약-1">요약</h2>
<h3 id="ip주소-네트워크-상의-호스트-식별">IP주소: 네트워크 상의 호스트 식별</h3>
<ul>
<li><strong>종류</strong>:<ul>
<li><strong>IPv4 (4바이트)</strong>: 현재 주력 주소 체계. 고갈 위기를 <strong>NAT</strong>(Network Address Translation) 기술로 극복 중.</li>
<li><strong>IPv6 (16바이트)</strong>: IPv4 주소 부족 문제를 근본적으로 해결하기 위해 등장한 차세대 체계.</li>
</ul>
</li>
<li><strong>클래스 분류 (A~E)</strong>:<ul>
<li><strong>목적</strong>: 조직 규모에 맞는 효율적 주소 할당 + 라우터의 빠른 주소 판독.</li>
<li><strong>핵심 구분</strong>: 첫 번째 바이트(8비트)의 <strong>시작 비트</strong> 모양새로 구분함.</li>
</ul>
</li>
</ul>
<h3 id="포트-번호-port-프로세스-식별-1">포트 번호 (PORT): 프로세스 식별</h3>
<ul>
<li><strong>정의</strong>: 하나의 IP(컴퓨터) 내에서 실행 중인 특정 프로그램(소켓)을 구분하는 번호.</li>
<li><strong>특징</strong>: 모든 통신 주소 뒤에는 포트가 숨어 있음 (예: HTTPS = 443).</li>
<li><strong>데이터 흐름</strong>: <strong>IP + PORT</strong> 정보를 기반으로 OS가 해당 데이터를 적절한 <strong>소켓(Socket)</strong>에 전달함.</li>
</ul>
<h3 id="핵심-구조체-sockaddr_in-1">핵심 구조체: <code>sockaddr_in</code></h3>
<ul>
<li><strong>용도</strong>: IPv4 주소 정보를 담는 표준 규격 봉투.</li>
<li><strong>구성</strong>: 주소 체계(<code>sin_family</code>), 포트 번호(<code>sin_port</code>), IP 주소(<code>sin_addr</code>), 패딩(<code>sin_zero</code>).</li>
<li><strong>주의</strong>: <code>sin_zero</code>는 반드시 <strong>0으로 초기화</strong>해야 함 (구조체 크기 맞춤용).</li>
</ul>
<h3 id="바이트-순서-변환-endianness-1">바이트 순서 변환 (Endianness)</h3>
<ul>
<li><strong>Host(CPU)</strong>: 보통 리틀 엔디안 (하위 바이트부터 저장).</li>
<li><strong>Network</strong>: 반드시 <strong>빅 엔디안</strong> (상위 바이트부터 저장, 표준).</li>
<li><strong>변환 함수</strong>:<ul>
<li><code>htons</code> / <code>ntohs</code>: 2바이트 포트 번호용 ($s$ = short)</li>
<li><code>htonl</code> / <code>ntohl</code>: 4바이트 IP 주소용 ($l$ = long)        </li>
</ul>
</li>
</ul>
<h3 id="주소-변환-함수-문자열-↔-정수-1">주소 변환 함수 (문자열 ↔ 정수)</h3>
<ul>
<li><strong><code>inet_aton</code></strong>: 문자열 IP(&quot;127.0.0.1&quot;)를 네트워크 바이트 순서 정수로 변환 (성공 시 1 반환).</li>
<li><strong><code>inet_ntoa</code></strong>: 네트워크 정수를 다시 문자열로 변환.<ul>
<li><strong>경고</strong>: 내부 정적 버퍼를 사용하여 재호출 시 값이 덮어씌워짐. 즉시 <code>strcpy</code>로 복사 필수.</li>
</ul>
</li>
<li><strong>현대적 대안</strong>: IPv6 대응 및 Thread-safe를 위해 실무에선 <strong><code>inet_pton</code></strong>, <strong><code>inet_ntop</code></strong> 권장.</li>
</ul>
